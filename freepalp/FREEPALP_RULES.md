---

## 1. Кодировка (КРИТИЧНО — Windows)

**ОБЯЗАТЕЛЬНО везде в проекте:**
- `open(..., encoding="utf-8")` — при любом чтении/записи файлов
- `subprocess`: НИКОГДА `text=True`. Всегда `capture_output=True` + ручной decode: `result.stdout.decode("utf-8", errors="replace")`
- `sys.stdout.reconfigure(encoding="utf-8")` — в точках входа (app.py, test_mvp.py)
- `json.dumps(..., ensure_ascii=False)` — для любого JSON с кириллицей
- Print с кириллицей — оборачивать через `_p()` или `encode("ascii", errors="replace")`
- Имена файлов, пути — только латиница, дефисы, подчёркивания

**ЗАПРЕЩЕНО:**
- `text=True` в subprocess (→ UnicodeDecodeError на Windows cp1251)
- Emoji в print() без гарантии UTF-8 окружения
- Хардкод кириллицы в f-строках которые пишутся в subprocess

## 2. Язык ответа

- Пользователь пишет по-русски → отвечать по-русски
- Пишет по-английски → отвечать по-английски
- Код, команды, технические термины → оставлять на оригинальном языке

---

## 3. Безопасность инструментов

**ЗАПРЕЩЕНО агенту:**
- Выполнять `rm -rf`, `del /f /s`, `format`, `DROP TABLE`
- Читать/писать файлы за пределами проекта (path traversal `../../`)
- Запускать команды не из whitelist `shell_tools.py`
- Хранить API ключи, пароли, токены в логах или памяти
- Записывать `.env`, `requirements.txt`, `setup.py` через инструменты

**РАЗРЕШЕНО — самомодификация:**
- `read_source(path)` — читать любой файл проекта для анализа
- `list_source(path)` — просматривать структуру исходников
- `write_source(path, content)` — изменять файлы в `freepalp/` для самоулучшения
- Всегда сначала `read_source` перед `write_source` — не терять существующий код

**ОБЯЗАТЕЛЬНО:**
- Файловые операции (рабочие файлы) — через `read_file`/`write_file` (sandbox)
- Анализ и улучшение системы — через `read_source`/`write_source`
- Все API ключи только в `.env`, никогда в коде
- Перед shell командой — проверять через ToolAgent whitelist

## 4. Self-Correction Loop

- Worker выполняет задачу
- Critic проверяет (score 0.0–1.0)
- Если `score < 0.7` или `PASSED: no` → обязательный retry
- Максимум 3 итерации (защита от зависания)
- При каждом retry — передавать полный feedback предыдущего Critic

---

## 5. Память и контекст

- HOT memory (`memory/hot_memory.md`) — максимум 100 строк, загружается в каждый промпт
- USER.md — контекст пользователя, обновляется автоматически при первом упоминании
- Активная версия ядра инжектируется в agent_memory каждой задачи: `[Активное ядро FreePalp: vX.X.X]`
- Каждая сессия → логируется в JSONL (`memory/sessions/`)
- Corrections.md — максимум 50 строк, старые архивируются
- Conversation history: последние 3 обмена инжектируются в контекст Worker'а

---

## 6. Провайдеры и fallback

- Приоритет: Ollama (local) → Groq (cloud) → OpenRouter → Cerebras → Gemini → SambaNova → GitHub
- Ollama проверяется через TCP `socket.create_connection(timeout=1)` — не HTTP (8 сек → 1 сек)
- Если провайдер недоступен → немедленный fallback в той же итерации
- Rate-limit → весь провайдер помечается как недоступный
- Не тратить итерацию на провайдер который уже упал

---

## 7. Трекинг токенов и стоимости

- Каждый вызов LLM: захватывать `usage.prompt_tokens` / `usage.completion_tokens`
- Логировать в `metrics.jsonl`: `tokens_in`, `tokens_out`, `cost_usd`
- Стоимость = `(tokens_total / 1000) * model.cost_per_1k`
- Отображать после каждой задачи: `[$$] Tokens: {in} in / {out} out | ~$X.XXXX`

---

## 8. Качество ответов

- Код всегда с type hints и docstring
- Обработка ошибок обязательна для IO операций
- Примеры использования для любой функции/класса
- Не придумывать факты — если неизвестно, сказать об этом явно

---

## 9. Самоулучшение

- `metrics.jsonl` обновляется после каждой задачи
- Каждые 5 задач (`AUTOIMPROVE_EVERY=5`) — автоматический цикл самоулучшения
- Новая версия активируется ТОЛЬКО если `test_mvp.py` показывает `6/6`
- Старая версия остаётся в `config/versions/v{N}/` как резервная
- Ручной откат: `/version rollback`

---

## 10. Конфиденциальность

- Не логировать полные API ключи
- SESSION JSONL не должны содержать credentials
- `_prompts_backup.json` автоматически удаляется после теста версии

## 11. Форматирование дат и времени (ISO 8601)

**ОБЯЗАТЕЛЬНО везде где используется дата/время:**
- Даты: `YYYY-MM-DD` (например: `2026-05-23`)
- Время: `HH:MM:SS` (например: `14:30:00`)
- Datetime: `YYYY-MM-DDTHH:MM:SS` (например: `2026-05-23T14:30:00`)
- `datetime.now().isoformat()` — стандарт для timestamp в JSON
- Временные промежутки: `timedelta`, не строки типа "2 дня"
- Вывод для пользователя: `strftime("%Y-%m-%d %H:%M")` — читаемо и стандартно

**ЗАПРЕЩЕНО:**
- `DD.MM.YYYY`, `MM/DD/YYYY` — региональные форматы ломают парсинг
- Строки типа "вчера", "2 дня назад" — не машиночитаемы

---

## 12. Переменные среды и конфиги

- `.env` — единственное место для API ключей
- `os.getenv("KEY", default)` — всегда с default чтобы не падать при отсутствии
- Никогда `os.environ["KEY"]` без try/except — KeyError → крэш
- `python-dotenv` загружается в точках входа (`load_dotenv(...)`)
