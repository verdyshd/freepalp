# FreePalp — эталон «обязан делать по умолчанию»

Суть FreePalp: бесплатный мульти-агентный оркестратор (Parser → Router → Worker → Critic)
поверх слабых/бесплатных моделей. Из сути следует главный принцип: **система не имеет права
полагаться на ум модели** — всё критичное должно гарантироваться детерминированной обвязкой.

Этот файл — эталон для тестов. Каждый пункт проверяем; провал = баг системы, не модели.

## 1. Честность выполнения (критично)
- [ ] Если задача подразумевает создание файла/папки — файл реально существует на диске после
      завершения. «Код в тексте ответа» = провал. Проверка детерминированная, не через критика.
- [ ] Никогда не сообщать «готово/сохранено», если инструмент не вызывался или вернул ошибку.
- [ ] Незавершённая после всех итераций задача → честное «не справился» + что успел, а не
      выдача сырого tool_call как «результата».

## 2. Идентичность и честность ответов
- [ ] На «кто ты / как тебя зовут» — всегда FreePalp, на любой модели-worker'е.
- [ ] «Не знаю» вместо выдумки; источники при фактических ответах не выдумывать.
- [ ] Память о пользователе содержит только реальные данные (никаких тестовых фикстур).

## 3. Понимание бытового языка
- [ ] Запросы обычного человека («сделай игру», «всё в одном файле», «положи в папку X»)
      классифицируются и исполняются без специальных формулировок и подсказок.
- [ ] Русский запрос → русский ответ; код и термины — на языке оригинала.

## 4. Кодинг
- [ ] Создаваемый код — запускаемый: HTML открывается, Python исполняется. Не псевдокод.
- [ ] Большая задача получает достаточный лимит токенов (классификация coding_large работает);
      обрезанный посреди строки файл = провал классификации, не модели.
- [ ] Многофайловый проект: умеет создать папку + несколько связанных файлов за одну задачу.

## 5. Работа с ПК
- [ ] Файловые операции строго в sandbox; выход за границы блокируется (path traversal).
- [ ] Shell — только whitelist, метасимволы блокированы.
- [ ] Действия видимы и проверяемы: пользователь может открыть результат в проводнике.

## 6. Чат-база (паритет с ChatGPT-минимумом)
- [ ] Контекст внутри сессии: помнит, о чём шла речь два сообщения назад.
- [ ] Память между сессиями: HOT/WARM/COLD реально подгружается и не содержит мусора.
- [ ] Отказ от вредных запросов; вежливая деградация при недоступности провайдеров.
- [ ] Форматирование: markdown, код в блоках, списки.

## 7. Управляемость человеком
- [ ] Пользователь может остановить агента в любой момент (CLI: Ctrl+C; WebUI: кнопка Stop);
      частичный прогресс при этом сохраняется, процесс не зависает.
- [ ] Видимый прогресс: на долгой задаче понятно, что происходит (итерация, модель, шаг).

## 8. Живучесть (следствие бесплатных провайдеров)
- [ ] Любой сетевой вызов имеет таймаут; зависание > N минут без вывода невозможно.
- [ ] Отказ провайдера → автоматический переход на следующего, не падение задачи.
- [ ] Критик дополняется детерминированными проверками везде, где провал типовой
      (файлы, идентичность, обрезанный вывод) — критик-LLM это пропускает.
