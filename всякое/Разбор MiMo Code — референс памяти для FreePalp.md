# MiMo Code — разбор архитектуры (референс для FreePalp)

Дата: 2026-06-11. Источники: техблог https://mimo.xiaomi.com/blog/mimo-code-long-horizon,
репозиторий https://github.com/XiaomiMiMo/MiMo-Code, строки бинарника v0.1.0.
Установлен локально: `F:\Tools\MiMo-Code\node_modules\.bin\mimo.cmd`.

## Что это
Форк OpenCode (терминальный кодинг-агент, TypeScript/Bun, MIT) + модели Xiaomi
(MiMo-V2.5-Pro, временно бесплатно). Вклад Xiaomi поверх OpenCode: подсистема
постоянной памяти, голосовой ввод, свои провайдеры, ACP.

## Цикл работы (стандартный агентный loop)
1. `mimo` в папке проекта → TUI; три режима-агента: build (полный доступ),
   plan (read-only), compose (по спецификации). Переключение Tab.
2. Промпт = системный + память проекта + инструменты → API модели → модель
   возвращает tool calls (read/write/bash/git/grep/LSP) → клиент выполняет
   локально → результат обратно в контекст → повтор до завершения.
3. Всё (сообщения, tool calls) пишется в SQLite (`~/.local/share/mimocode/mimocode.db`)
   с FTS5-полнотекстовым поиском.

## Управление контекстом — главное, что стоит заимствовать
- Окно делится на «циклы». Чекпойнты срабатывают на ~20%, 45%, 70% бюджета окна —
  НЕ при переполнении, а заранее и инкрементально.
- Параллельно с основным агентом работает **writer-субагент**: ведёт checkpoint.md
  с фиксированной структурой из 11 полей. Основной агент не тратит на это ход.
- При приближении к лимиту — rebuild: новое окно, слоёный промпт с независимым
  токен-бюджетом на каждый слой, суммарно ≤65K токенов. Порядок впрыска:
  задача → чекпойнт сессии → сообщения пользователя → память проекта → прочее.
- Итог: «бесконечная» сессия = реконструкция с потерями, но потери управляемые
  (фиксированная структура чекпойнта, приоритет задачи и слов пользователя).

## Четыре слоя памяти
1. checkpoint.md — рабочее состояние текущей сессии (11 полей).
2. MEMORY.md — постоянные знания о проекте.
3. Глобальные предпочтения пользователя (кросс-проектные).
4. SQLite-трейс всей истории (сообщения + tool calls) с FTS5-поиском —
   агент может искать по своей истории как по базе.

## Фоновая консолидация
- **Dream** (раз в 7 дней): дедупликация и сжатие памяти, удаление устаревшего.
- **Distill** (раз в 30 дней): поиск повторяющихся паттернов в истории →
  генерация переиспользуемых «скиллов» (формат совместим со скиллами Claude Code).
- Оба — независимые агенты, не основной.

## Идеи для FreePalp (кандидаты)
- [ ] Триггеры чекпойнтов по % окна (20/45/70), а не по факту переполнения.
- [ ] Отдельный дешёвый writer-агент для ведения состояния параллельно Worker'у
      (у нас уже есть роли — это четвёртая: Parser/Router/Worker/Critic + Writer).
- [ ] Фиксированная схема чекпойнта (их 11 полей) вместо свободного текста —
      предсказуемая реконструкция.
- [ ] Слоёный rebuild с токен-бюджетом на слой и явным порядком приоритетов.
- [ ] FTS5-поиск по собственной истории сессий (SQLite уже умеет из коробки).
- [ ] Аналог Distill: периодическая выжимка повторяющихся цепочек tool calls
      в готовые процедуры.

## Минусы / предупреждения
- Бенчмарки «лучше Claude Code» — вендорские, в день релиза, без независимой проверки.
- При первом запуске молча автоимпортирует сессии Claude Code из ~/.claude/projects
  в свою базу (отключение: MIMOCODE_DISABLE_CLAUDE_IMPORT=1).
- Бесплатная модель = промпты и код уходят на серверы Xiaomi, действует их ToS.
