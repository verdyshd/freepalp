# Odysseus (PewDiePie) — разбор и сравнение с FreePalp

Дата: 2026-06-11. Репо: https://github.com/pewdiepie-archdaemon/odysseus
(68k звёзд, 1145 коммитов, AGPL-3.0, автор — реально PewDiePie/Felix Kjellberg).
Склонирован: `F:\Tools\odysseus` (~35 МБ, Python 49% + JS 40%).

## Что это
Self-hosted персональный AI-воркспейс: веб-приложение (FastAPI + свой JS-фронт, PWA),
замена ChatGPT/Claude.ai на своём железе. Локал-первый: vLLM/llama.cpp/Ollama +
облачные API (OpenRouter, OpenAI, Copilot). Не оркестратор и не кодинг-агент —
«операционка для личного ИИ»: чат, агент с инструментами, почта/календарь/контакты
(CalDAV), заметки, документы, галерея, TTS/STT, deep research, планировщик задач.

## Архитектура (ключевые файлы)
- `src/agent_loop.py` (186K) — одиночный агентный цикл, multi-round.
  Tool calls ДВУМЯ протоколами: нативный function calling И fenced-блоки в тексте —
  работает даже с локальными моделями без поддержки tool calling.
- `src/tool_implementations.py` (187K) + `tool_schemas.py` (83K) — десятки инструментов
  (shell, файлы, web, память, документы) + любой MCP.
- `src/context_compactor.py` — компакция на 85% окна, саммари той же моделью (≤1024 ток).
- `services/memory/` — память: FAISS/ChromaDB векторы + фоновый экстрактор фактов
  после каждого ответа + периодический LLM-аудит (дедупликация). Хитрость: fingerprint
  состояния памяти — аудит скипается, если ничего не менялось (экономия 30–120с/вызов).
- `src/teacher_escalation.py` — ГЛАВНАЯ находка, см. ниже.
- `services/hwfit/` — подбор моделей под железо пользователя (GPU/RAM).
- `routes/cookbook_routes.py` (161K) — discovery и сервинг локальных моделей.
- Безопасность всерьёз: THREAT_MODEL.md, prompt_security, tool_policy, url_safety,
  loop breaker от зацикливания, тесты на prompt injection через skill index.
- Skills в формате SKILL.md — совместимы с Claude Code; есть интеграции-плагины
  для Claude Code и Codex (integrations/).

## Teacher escalation (учитель-ученик) — идея №1 для заимствования
Когда локальный «студент» завершает ход в агент-режиме:
1. Tier 1: дешёвая regex-проверка вывода на признаки провала («I don't have a tool»,
   «Unknown action» и т.п.). Бесплатно, мгновенно.
2. Провал → полный контекст уходит SOTA-«учителю» (настраиваемый облачный API),
   тот даёт корректирующий ответ И ПИШЕТ SKILL.md — процедуру, чтобы студент
   в следующий раз справился сам.
3. Скилл сохраняется только если ответ учителя сам проходит ту же проверку.
4. Эскалация выключена, если основная модель и так SOTA (список хостов).
Итог: система дистиллирует умения дорогой модели в скиллы для дешёвой — улучшение
накапливается, а не сгорает с ответом.

## Сравнение с FreePalp
Общее: мульти-провайдер, ReAct-цикл (~40 инструментов у обоих), память с консолидацией,
самокоррекция. Различия по сути:

| | FreePalp | Odysseus |
|---|---|---|
| Категория | оркестратор задач (пайплайн) | персональный воркспейс (продукт с UI) |
| Выбор модели | Router: автоматически под тип задачи, 40+ моделей | пользователь руками; hwfit — только под железо |
| Самокоррекция | Critic оценивает КАЖДЫЙ результат (0–1), retry ≤3 | teacher только при детектированном провале локалки |
| Куда идёт коррекция | в повтор текущей задачи (сгорает) | в постоянный SKILL.md (накапливается) |
| Планирование | Architect строит DAG | нет (одиночный loop + планировщик по расписанию) |
| Зрелость | исследовательский | продукт: 1145 коммитов, threat model, толпа тестов |

FreePalp сильнее в: роутинге по моделям, DAG-декомпозиции, систематическом контроле
качества. Odysseus сильнее в: накоплении улучшений (teacher→skill), инженерной
зрелости (тесты, безопасность), работе с локальными моделями.

## Чеклист идей для FreePalp
- [ ] **Teacher→skill дистилляция**: Critic при провале не просто ретраит, а после
      успешного ретрая сильной моделью сохраняет процедуру (скилл/память), чтобы
      дешёвая модель справлялась сама. Сейчас коррекции FreePalp сгорают.
- [ ] **Двухъярусная детекция провала**: сначала бесплатный regex по типовым
      признакам, LLM-оценка — только для неоднозначного. Сейчас Critic жжёт токены
      на каждом результате.
- [ ] **Fenced-блоки как fallback-протокол tool calls** — у бесплатных провайдеров
      function calling часто кривой; текстовый протокол спасает.
- [ ] **Loop breaker**: детектор зацикливания агента (повтор одинаковых tool calls).
- [ ] **Fingerprint-скип консолидации памяти**: не гонять LLM-аудит, если память
      не менялась с прошлого раза.
- [ ] Тесты на prompt injection через содержимое памяти/скиллов (у них есть, у нас?).
